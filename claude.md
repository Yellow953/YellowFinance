# YellowFinance вЂ” Flutter App

## Project Overview
A personal finance mobile app with expense/income tracking, AI-powered financial analysis, and an investing section (stocks, ETFs, crypto, gold/silver). Built with Flutter, Firebase backend, GetX state management, and MVC architecture.

---

## Tech Stack

| Layer | Technology |
|---|---|
| Framework | Flutter (latest stable) |
| State Management | GetX |
| Architecture | MVC (Model / View / Controller) |
| Backend | Firebase (Firestore, Auth, Storage, Functions) |
| AI Analysis | Google Gemini |
| Market Data | CoinGecko API (via Firebase Functions) |
| Local Storage | flutter_secure_storage (sensitive), shared_preferences (non-sensitive) |
| Charts | fl_chart |

### Firestore Security Rules
```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /users/{userId}/{document=**} {
      allow read, write: if request.auth != null && request.auth.uid == userId;
    }
  }
}
```

## Security Requirements

### Authentication
- Email/password with email verification required before access
- Google Sign-In as secondary option
- Session persists via Firebase Auth (no manual token storage)
- Auto sign-out after 30 days of inactivity

### Data Security
- All API keys (Claude, market data) live ONLY in Firebase Functions environment вЂ” never in Flutter code or pubspec
- Use `flutter_secure_storage` for any locally cached sensitive data (never SharedPreferences for tokens)
- Enable Firebase App Check (Play Integrity on Android, DeviceCheck on iOS)
- All Firestore reads/writes gated by auth UID matching (see rules above)
- Input validation on all user-entered amounts, descriptions, symbols
- No PII logged to console in production вЂ” use `kDebugMode` guards

### Network
- Certificate pinning via `dio` interceptors for Firebase Functions calls
- All market data and AI calls go through Firebase Functions (not direct from app)

### Build
- Obfuscate release builds: `flutter build apk --obfuscate --split-debug-info=build/debug-info`
- ProGuard rules for Android release

## GetX Architecture Rules

### Controllers
- One controller per module
- Controllers extend `GetxController`
- Use `.obs` for all reactive state
- Dispose streams and subscriptions in `onClose()`
- Never put business logic in Views

```dart
// Example pattern
class HomeController extends GetxController {
  final TransactionRepository _txnRepo;
  final RxList<TransactionModel> transactions = <TransactionModel>[].obs;
  final RxDouble totalBalance = 0.0.obs;
  final RxBool isLoading = false.obs;

  HomeController({required TransactionRepository txnRepo})
      : _txnRepo = txnRepo;

  @override
  void onInit() {
    super.onInit();
    loadDashboard();
  }

  Future<void> loadDashboard() async { ... }
}
```

### Bindings
- Every route has a binding that injects its controller and dependencies
- Use `Get.lazyPut` for controllers, `Get.put` for services in main

### Routes
- All navigation via `Get.toNamed()` вЂ” no direct Navigator calls
- Route guards via `GetMiddleware` for auth protection

## Firebase Cloud Functions (Node.js)

Create a `/functions` directory alongside `/lib`.

### Required Functions

```
functions/
в”њв”Ђв”Ђ index.js
в”њв”Ђв”Ђ ai/
в”‚   в””в”Ђв”Ђ analyzeFinances.js     # Calls Gemini API with user financial context
в””в”Ђв”Ђ market/
    в””в”Ђв”Ђ fetchPrices.js         # Fetches live prices for portfolio assets
```

### analyzeFinances function
- Accepts: `{ userId, prompt }` (callable function)
- Fetches last 90 days of transactions from Firestore
- Builds context summary (income, expenses, categories, portfolio)
- Calls Gemini API with context + user prompt
- Returns AI response text
- Rate limit: 20 calls/user/day stored in Firestore

### fetchPrices function
- Scheduled: runs every 15 minutes
- Fetches prices for all unique symbols in all user portfolios
- Writes to `market_prices/{symbol}` collection
- Flutter app reads from this collection (no direct API calls)

## Implementation Order

Build in this sequence to avoid dependency issues:

1. **Foundation** вЂ” Firebase setup, `firebase_options.dart`, core constants (colors, text styles, theme)
2. **Auth module** вЂ” Login, Register, Forgot Password views + controller + Firebase Auth integration
3. **Main layout** вЂ” Bottom nav bar, route shell, auth guard middleware
4. **Home module** вЂ” Balance hero card, quick actions grid, recent transactions list
5. **Transactions module** вЂ” Transaction list, add/edit transaction bottom sheet
6. **Portfolio module** вЂ” Asset list, filter chips, portfolio balance card
7. **Reports module** вЂ” Monthly chart (fl_chart bar), category breakdown (pie chart)
8. **AI module** вЂ” Chat UI, Firebase Function call, context injection
9. **Security hardening** вЂ” App Check, secure storage audit, release obfuscation config
10. **Firebase Functions** вЂ” Deploy analyzeFinances + fetchPrices functions

## Code Quality Rules

- Run `flutter analyze` with zero warnings before committing
- All public methods and classes have doc comments
- No hardcoded strings вЂ” use `AppConstants` or localization keys
- No hardcoded colors вЂ” always use `AppColors.*`
